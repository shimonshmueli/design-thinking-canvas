2.7 Value Proposition价值主张

What it is

A sharp statement of the value the eventual solution must deliver: for whom, what gain it creates or what pain it removes, and why it beats the alternatives. Drafting it in Define — before any solution exists — is a forcing function: if you can't say what value the solution must create, the problem isn't defined yet.一句精炼的陈述，说明最终方案必须交付的价值：为谁而做、创造了什么收益或消除了什么痛点，以及为什么它优于其他替代方案。在定义阶段就起草它——此时还没有任何具体方案——是一种强制机制：如果你说不清方案必须创造什么价值，那说明问题本身还没有定义清楚。

How to use it

  1. Anchor it to persona needs: which pains are severe and frequent enough to be worth solving?把它锚定在用户画像的需求上：哪些痛点足够严重、足够频繁，值得去解决？
  2. State the alternative users have today, including doing nothing.说明用户目前拥有的替代选项，包括『什么都不做』这一选项。
  3. Draft the proposition as a testable claim, and mark assumptions to verify in Evaluate.把价值主张写成一个可检验的论断，并标出需要在评估阶段验证的假设。
  4. Keep it solution-agnostic: value to deliver, not features to build.保持与具体方案无关：写的是要交付的价值，而不是要构建的功能。

Watch out for

Propositions that describe your intent ('we will build a platform that…') instead of the user's gain. If a user wouldn't nod at it, rewrite it.警惕那些描述你自己意图（『我们将打造一个……的平台』）而非用户收益的价值主张。如果用户听了不会点头认同，就要重写它。

Worksheet

Apply Value Proposition to your project and capture what you find. Entries are saved in your browser, appear under "From tool worksheets" on the Define page, feed the AI context there, and can be sent up as Define stage cards.运用「价值主张」处理你的项目，并记录你的发现。条目会保存在你的浏览器中，显示在「定义」页面『来自工具工作表』区块下，并纳入该页面的 AI 上下文，还可以被发送为「定义」阶段的卡片。
