Value Proposition

What it is

A sharp statement of the value the eventual solution must deliver: for whom, what gain it creates or what pain it removes, and why it beats the alternatives. Drafting it in Define — before any solution exists — is a forcing function: if you can't say what value the solution must create, the problem isn't defined yet.

How to use it

  1. Anchor it to persona needs: which pains are severe and frequent enough to be worth solving?
  2. State the alternative users have today, including doing nothing.
  3. Draft the proposition as a testable claim, and mark assumptions to verify in Evaluate.
  4. Keep it solution-agnostic: value to deliver, not features to build.

Watch out for

Propositions that describe your intent ('we will build a platform that…') instead of the user's gain. If a user wouldn't nod at it, rewrite it.

Worksheet

Apply Value Proposition to your project and capture what you find. Entries are saved in your browser, appear under "From tool worksheets" on the Define page, feed the AI context there, and can be sent up as Define stage cards.
