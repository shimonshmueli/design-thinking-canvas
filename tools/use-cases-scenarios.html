2.4 Use Cases & User Scenarios用例与用户场景

↓ Jump to workspace

What it is

Narrative descriptions of how a user accomplishes a goal: use cases enumerate the interactions between an actor and a system to achieve an outcome; scenarios tell the richer story around them — context, motivation, interruptions. They translate the abstract problem into concrete situations a solution will have to handle.以叙事方式描述用户如何达成某个目标：用例列举出行为者与系统之间为实现某个结果而产生的交互；场景则讲述围绕这些交互展开的更丰富的故事——情境、动机、中断等。它们把抽象的问题转化为解决方案必须应对的具体情境。

How to use it

  1. Start from persona goals: what is this person trying to get done?从用户画像的目标出发：这个人想要完成什么？
  2. Write the main success path first, then the variations and failure paths.先写出主要的成功路径，再写各种变体与失败路径。
  3. Keep scenarios grounded in researched context, including the messy parts.让场景始终扎根于调研得到的真实情境，包括其中混乱的部分。
  4. Use them to pressure-test the problem statement: does it cover these situations?用它们来压力测试问题陈述：问题陈述是否覆盖了这些情境？

Watch out for

Writing only the happy path. The edge cases and interruptions are usually where the real design problem lives.只写『理想路径』是不够的。边缘情况与中断场景，往往才是真正设计问题所在之处。

Worksheet

Apply Use Cases & User Scenarios to your project and capture what you find. Entries are saved in your browser, appear under "From tool worksheets" on the Define page, feed the AI context there, and can be sent up as Define stage cards.运用「用例与用户场景」处理你的项目，并记录你的发现。条目会保存在你的浏览器中，显示在「定义」页面『来自工具工作表』区块下，并纳入该页面的 AI 上下文，还可以被发送为「定义」阶段的卡片。
