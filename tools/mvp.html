Minimum Viable Product (MVP)

What it is

The smallest version of a solution that lets you test its core value hypothesis with real users at real stakes. An MVP is a learning instrument, not a small product: it exists to answer 'do people actually want and use this?' with evidence, before major investment.

How to use it

  1. State the riskiest assumption in the value proposition.
  2. Design the smallest thing that tests exactly that assumption — it may be a concierge service, a landing page, or a single-feature build.
  3. Define success metrics before launch.
  4. Ship to real users, measure, and decide: persevere, pivot, or kill.

Watch out for

The 'small first release' misreading — cramming a full roadmap into v1 and calling it an MVP. If it doesn't test a named assumption, it's not one.
